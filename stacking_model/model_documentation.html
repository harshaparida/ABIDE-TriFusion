Key Discovery

The Most Important Finding

Keeping FC features RAW (not harmonized) was critical for performance.

When FC features were harmonized using ComBat, accuracy dropped to ~60%. By keeping them raw, LogisticRegression alone achieved 75.84% accuracy, and the final stacked model achieved 77.22%.

Enhanced Stacking Model Architecture

# ================================================ # LEVEL 1: Base Models (3-Fold CV with Feature Selection) # ================================================ for each fold: # SelectKBest with f_classif inside CV (NO data leakage) X_selected = SelectKBest(k=2500).fit_transform(X_train, y_train) # Model 1: XGBoost xgb = XGBClassifier( n_estimators=500, max_depth=5, learning_rate=0.02, scale_pos_weight=class_ratio, colsample_bytree=0.8 ) # Model 2: LightGBM lgbm = LGBMClassifier( n_estimators=500, num_leaves=31, learning_rate=0.02, is_unbalance=True ) # Model 3: Logistic Regression lr = LogisticRegression( C=1.0, class_weight='balanced', max_iter=500 ) # Generate OOF predictions P1 = [xgb_proba, lgbm_proba, lr_proba] # ================================================ # LEVEL 2: Meta Learners (on Level 1 OOF predictions) # ================================================ P2 = [xgb_meta, lgbm_meta, lr_meta] # ================================================ # LEVEL 3: Feature Interactions # ================================================ # Create interaction features from Level 2 predictions interactions = [ P2[0] * P2[1], # xgb * lgbm P2[0] * P2[2], # xgb * lr P2[1] * P2[2], # lgbm * lr (P2[0] + P2[1])/2, # mean |P2[0] - P2[1]| # abs diff ] for each fold: P3 = XGBClassifier(max_depth=3, n_estimators=100) # ================================================ # FINAL: Combine All Levels # ================================================ final_proba = (P1.mean() + P2.mean() + P3) / 3.0 prediction = (final_proba >= threshold).astype(int)

Key Discoveries & Solutions

Problem: Harmonization Hurt Performance

Issue: FC features harmonized with ComBat resulted in ~60% accuracy (barely better than random).

Solution: Keep FC features raw, only harmonize sMRI features.

Problem: Feature Selection Location

Issue: Applying SelectKBest BEFORE cross-validation caused data leakage and inflated metrics.

Solution: Move SelectKBest INSIDE the CV loop to prevent leakage.

Problem: Too Many/Different Models

Issue: Using many different model types and hyperparameters without proper CV feature selection led to overfitting.

Solution: Use consistent feature selection inside CV with XGBoost, LightGBM, and LogisticRegression.

Success: LogisticRegression Was Key

LogisticRegression with class balancing achieved 75.84% accuracy - the best single model. The stacking combined it with tree-based models for the final 77.22%.

Summary

What Made the Difference?

  1. Keep FC features raw - Harmonization removed discriminative information
  2. Variance-based pre-selection - Top 6500 features from 8111
  3. SelectKBest inside CV - Prevents data leakage, selects relevant features per fold
  4. LogisticRegression - Best single model, works well with many features
  5. Multilevel stacking - Combines tree-based and linear models effectively
  6. Feature interactions - Level 3 captures non-linear relationships between model predictions
